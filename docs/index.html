Model Lab — does advanced ML beat a good Elo model?

Four forecasters trained on 19,673 internationals (2002–2022) and tested on 3,694 matches they never saw (2023–2026, including this World Cup). Same features, same metrics — the honest way to choose a model, not the flashiest one.

The finding. The advanced models beat the Elo baseline by under 1% in log-loss, and Elo keeps the best raw accuracy (60.7%). With only ~64 World-Cup matches of signal, sophistication buys almost nothing — so the headline forecast stays on the interpretable Elo + Monte Carlo model, now validated against a Dixon-Coles Poisson model, gradient boosting and a neural network. Choosing the simpler model because the evidence says so is the senior move.

Methodology & data

Data. Historical results from the Fjelstul World Cup Database (all 22 tournaments, 1930–2022: matches, goals, squads, awards) and martj42/international_results (49,000+ internationals since 1872, updated daily — includes the ongoing 2026 World Cup).

Ratings. Elo computed over the full match history: K = 60 for World Cup matches down to 20 for friendlies, scaled by goal margin, +100 home advantage for non-neutral venues.

Forecast. The confirmed quarter-final bracket is simulated 10,000 times; each tie is resolved with the logistic Elo win expectancy P(A) = 1 / (1 + 10−ΔElo/400). Probabilities are the share of simulations in which a team reaches each round.

Validation (Model Lab). The Elo model is not taken on faith: it is benchmarked against a Dixon-Coles Poisson goals model, gradient boosting (XGBoost) and a neural network (MLP), all trained on 19,673 internationals (2002–2022) and scored on 3,694 matches they never saw (2023–2026). On the hold-out the advanced models improve log-loss by under 1% and Elo keeps the best accuracy — so the interpretable model is kept by evidence, not habit. Features (Elo gap, form, goal rates, rest, venue, tournament weight) are all computed pre-match to avoid leakage.

Stack. Python — pandas / NumPy for the pipeline, scikit-learn + XGBoost + SciPy for the models — → JSON → this dashboard (Apache ECharts). Reproducible: python src/features.py && python src/train_models.py && python src/simulate.py.
